## Questions to Ask First

Before designing any system, gather this information:

1. **What is your current biggest frustration?** (skipping tasks? Procrastinating? Overwhelm? Context switching? Finishing the wrong things?)
2. **What tools do you already use?** (Paper notebook, digital app, calendar, sticky notes, nothing?)
3. **What is your work type?** (Knowledge work, creative, managerial, mixed, physical + cognitive?)
4. **How much of your day do you control?** (Fully self-directed, mostly meetings, reactive/on-call, hybrid?)
5. **Have you tried any productivity systems before? What failed and why?**
6. **Do you have ADHD, executive function challenges, or energy fluctuations that affect planning?**
7. **What does a successful day look like for you?**

## The Adaptive Productivity Stack

Our framework synthesizes the best principles from five major methodologies into a modular, pick-and-choose system. No single methodology works for everyone because people have different failure modes. The Adaptive Productivity Stack addresses this by organizing productivity into five layers, each targeting a specific failure mode.

### The Five Layers

```
Layer 5: EXECUTION ENGINE    - How you do the work (focus techniques)
Layer 4: SCHEDULE ARCHITECT  - When work gets done (time allocation)
Layer 3: PRIORITY FILTER     - What matters most (decision framework)
Layer 2: PROCESSING SYSTEM   - How inputs become actions (clarification)
Layer 1: CAPTURE NET         - Nothing falls through the cracks (collection)
```

Each layer draws from the methodology that handles that function best, but you only need the layers where you currently struggle.

## Source Methodology Comparison

| Approach | Best For | Key Insight | Limitation |
|----------|----------|-------------|------------|
| GTD (David Allen) | Knowledge workers with high input volume | Your mind is for having ideas, not holding them - externalize everything | Complex setup; can become system-maintenance overhead |
| Bullet Journal (Ryder Carroll) | People who think on paper; need flexibility | Rapid logging + monthly migration forces regular review of commitments | Analog-only by design; hard to search; no collaboration |
| Time Blocking (Cal Newport) | Makers who need deep work protection | Every minute should have a job; intention beats reaction | Rigid; breaks down with unpredictable schedules |
| Pomodoro Technique (Francesco Cirillo) | Procrastinators; people who struggle to start | Bounded time chunks lower the activation energy to begin | Artificial breaks disrupt flow; not ideal for all task types |
| Eisenhower Matrix (Eisenhower/Covey) | Leaders drowning in urgent-but-unimportant work | Distinguish urgent from important; most people neglect Quadrant 2 | Binary categories oversimplify; does not help with scheduling |

## Layer 1: The Capture Net

**Problem it solves:** Things fall through the cracks. You overlook tasks, ideas, and commitments.

**Core principle (from GTD):** Your brain is unreliable storage. Every open loop must be captured in a trusted external system within seconds of occurring.

### Capture Rules

1. **Ubiquitous capture** - You must have a capture tool within arm's reach at all times
2. **Zero friction** - If capture takes more than 10 seconds, the system will fail
3. **One inbox maximum three** - Consolidation prevents lost items (one physical, one digital, one voice)
4. **Capture is NOT processing** - Write the raw thought; do not organize yet

### Capture Tool Selection

| Context | Recommended Tool | Why |
|---------|-----------------|-----|
| At desk | Open text file or app inbox | Fastest typing |
| In transit | Phone voice memo or quick-entry app | Low friction |
| In meetings | Notebook rapid-log (Bullet Journal style) | Non-disruptive |
| In bed/shower | Waterproof notepad or voice assistant | Capture before sleep erases |

### The Bullet Journal Rapid Log Format

For paper capture, use rapid logging symbols:

```
*  Task
-  Note
o  Event
!  Inspiration/Idea
>  Migrated forward
<  Scheduled to calendar
X  Completed
~  Cancelled (with brief reason)
```

This notation lets you capture fast and process later.

## Layer 2: The Processing System

**Problem it solves:** You have a pile of captured stuff but do not know what to do with it.

**Core principle (from GTD):** Every captured item must be processed with a clear decision: Do it, Delegate it, Defer it, File it, or Delete it.

### The Processing Flowchart

```
For each captured item, ask:

1. Is this actionable?
   NO  -> Reference material (file it) or trash (delete it)
   YES -> Continue

2. Will it take less than 2 minutes?
   YES -> Do it right now
   NO  -> Continue

3. Am I the right person?
   NO  -> Delegate it (and track the delegation)
   YES -> Continue

4. Does it have a hard deadline or specific date?
   YES -> Schedule it (goes to Layer 4)
   NO  -> Add to prioritized action list (goes to Layer 3)
```

### Processing Cadence

- **Daily processing** (10 minutes): Clear all inboxes to zero
- **Weekly review** (30-60 minutes): Review all projects, lists, and calendar; update priorities
- **Monthly migration** (from Bullet Journal): Review all open items; migrate what still matters; cancel what does not

The monthly migration is powerful because it forces you to re-commit to every open task. Items you repeatedly skip migrating should be cancelled.

## Layer 3: The Priority Filter

**Problem it solves:** Everything feels equally important. You work on whatever is loudest.

**Core principle (from Eisenhower Matrix):** Urgency and importance are different axes. Most people spend all day on urgent-but-unimportant tasks while neglecting important-but-not-urgent work.

### The Four-Quadrant Assessment

```
                    URGENT              NOT URGENT
            +-------------------+-------------------+
 IMPORTANT  | Q1: CRISIS        | Q2: STRATEGIC     |
            | Do immediately    | Schedule dedicated |
            | (fires, deadlines)| time blocks        |
            +-------------------+-------------------+
 NOT        | Q3: INTERRUPTION  | Q4: NOISE         |
 IMPORTANT  | Delegate or batch | Eliminate or limit |
            | (most emails,     | (social media,     |
            |  some meetings)   |  busywork)         |
            +-------------------+-------------------+
```

### Priority Scoring (Our Enhancement)

For your daily task list, score each item:

```
Impact Score (1-5):  How much does completing this move a meaningful goal forward?
Urgency Score (1-5): What is the cost of delay?
Effort Score (1-5):  How much energy/time does it require?

Priority = (Impact x 2 + Urgency) / Effort
```

High-priority items have high impact, reasonable urgency, and manageable effort. This formula naturally surfaces quick wins and strategic work while deprioritizing low-impact busywork.

### The Daily Big Three

Each morning (or the night before), select exactly three tasks that would make the day a success. These are your non-negotiable commitments. Everything else is bonus.

## Layer 4: The Schedule Architect

**Problem it solves:** You know what matters but never find time for it. Your calendar is reactive.

**Core principle (from Time Blocking):** A task without a time slot is a wish. Block time for your priorities before others fill your calendar.

### Time Block Categories

```
[DEEP]    - Focused, cognitively demanding work (2-4 hour blocks)
[SHALLOW] - Administrative, email, routine tasks (batch into 30-60 min blocks)
[MEETING] - Collaborative time (cluster to minimize context switching)
[BUFFER]  - Overflow and unexpected tasks (15-30 min between blocks)
[RECHARGE]- Breaks, movement, meals (non-negotiable)
```

### The Ideal Week Template

Design your ideal week before the week starts:

```
Monday:    [DEEP 9-12] [SHALLOW 12-1] [MEETING 2-4] [BUFFER 4-4:30]
Tuesday:   [DEEP 9-11] [MEETING 11-12] [DEEP 1-3] [SHALLOW 3-4:30]
Wednesday: [DEEP 9-12] [SHALLOW 12-1] [DEEP 2-4] [BUFFER 4-4:30]
Thursday:  [MEETING 9-12] [SHALLOW 12-1] [DEEP 1-3] [BUFFER 3-4:30]
Friday:    [DEEP 9-11] [SHALLOW 11-12] [WEEKLY REVIEW 1-2] [BUFFER 2-3]
```

This template is aspirational. Actual weeks will deviate. The value is in having a default structure rather than starting each week from zero.

### Energy-Aware Scheduling

Place your most demanding work during your peak energy hours. Most people have a cognitive peak in late morning (9-12) and a secondary peak in mid-afternoon (3-5). Schedule deep work there. Schedule shallow work during your post-lunch dip.

## Layer 5: The Execution Engine

**Problem it solves:** You sit down to work and cannot start, get distracted, or cannot sustain focus.

**Core principle (from Pomodoro Technique):** Bounded work intervals with defined breaks reduce resistance to starting and prevent burnout.

### The Focus Sprint Protocol

```
1. CHOOSE   - Select one task from your Big Three
2. SET      - Set a timer (25 min standard, adjust 15-50 based on task)
3. COMMIT   - Close all unrelated tabs, silence phone, signal "do not disturb"
4. WORK     - Single-task until the timer rings
5. MARK     - Record what you accomplished
6. BREAK    - 5 min short break (stand, stretch, hydrate)
7. REPEAT   - After 4 sprints, take a 15-30 min long break
```

### When NOT to Use Timed Sprints

Pomodoro-style sprints are excellent for tasks with high resistance or low inherent interest. But when you are in a flow state on creative or complex work, do NOT break your focus because a timer rang. The timer is a tool to start focus, not to interrupt it.

**Adapt the rule:** If the timer rings and you are in deep flow, reset for another full sprint without a break. Take the break when the flow naturally ebbs.

### Distraction Logging

Keep a notepad next to you during focus sprints. When an unrelated thought intrudes (email to send, thing to look up, idea), jot it on the notepad and return to work. Process these captures later. This acknowledges the thought without acting on it.

## Build Your Personal System

### Step 1: Identify Your Failure Mode

```
"I overlook things and drop balls"          -> Start with Layer 1 (Capture)
"I have huge lists but never process"     -> Start with Layer 2 (Processing)
"I work hard on the wrong things"         -> Start with Layer 3 (Priority)
"Important work never gets time"          -> Start with Layer 4 (Scheduling)
"I sit down to work and can't focus"      -> Start with Layer 5 (Execution)
"Everything is a mess"                    -> Start with Layer 1, add one layer per week
```

### Step 2: Start with ONE Layer

Do not implement all five layers at once. That is the most common reason productivity systems fail: too much system overhead. Pick the one layer that addresses your biggest pain point. Use it for two weeks. Then add the next layer.

### Step 3: Choose Your Tools

| Preference | Capture | Lists | Calendar | Timer |
|-----------|---------|-------|----------|-------|
| All analog | Notebook (BuJo) | Same notebook | Paper planner | Kitchen timer |
| All digital | Quick-entry app | Task manager | Digital calendar | Phone timer |
| Hybrid | Pocket notebook + app | Digital task manager | Digital calendar | Any timer |

### Step 4: Set Your Review Cadence

- **Daily** (5-10 min): Process inboxes, set tomorrow's Big Three
- **Weekly** (30-60 min): Review all projects, clear stale tasks, plan next week's time blocks
- **Monthly** (60-90 min): Review goals, migrate/cancel old tasks, adjust system

### Step 5: Iterate

No system survives first contact with real life unchanged. After two weeks with each layer, ask:

1. What is working that I should keep?
2. What is creating friction that I should simplify?
3. What gap remains that needs the next layer?

## Common Failure Patterns and Fixes

| Failure | Root Cause | Fix |
|---------|-----------|-----|
| System abandoned after 2 weeks | Too complex; too many layers at once | Strip to one layer; rebuild |
| Lists grow forever | No processing or review habit | Set a daily 10-min processing alarm |
| Calendar ignored | Time blocks too rigid for real life | Add buffer blocks; use "ideal week" as guide not law |
| Still procrastinating despite system | Emotional resistance, not organizational | Use Layer 5 sprints; break tasks smaller; address underlying avoidance |
| Spending more time on system than work | Over-engineering; tool addiction | Simplify ruthlessly; the system serves the work |

## Adaptation for ADHD and Executive Function Challenges

If executive function is a challenge, modify the stack:

1. **Capture**: Use voice memos (lowest friction)
2. **Processing**: Process items immediately when possible rather than batching (out of sight = out of mind)
3. **Priority**: Limit to ONE priority per day, not three
4. **Scheduling**: Use visual timers and body-doubling
5. **Execution**: Shorter sprints (15 min); more frequent breaks; novelty rotation between tasks

## Further Reading

For deeper exploration of the source methodologies:

- **Getting Things Done** by David Allen - The definitive capture-and-process system
- **The Bullet Journal Method** by Ryder Carroll - Analog intentionality and rapid logging
- **Deep Work** by Cal Newport - The case for time blocking and focus protection
- **The Pomodoro Technique** by Francesco Cirillo - Bounded work intervals for focus
- **The 7 Habits of Highly Effective People** by Stephen Covey - The Eisenhower Matrix popularized as "Quadrant 2" thinking

Each of these provides deeper detail on its respective layer. The Adaptive Productivity Stack gives you the integration layer that no single methodology provides on its own.
